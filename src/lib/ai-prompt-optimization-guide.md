## 二、最佳实践指南

### 1. 提示词结构优化

#### 1.1 结构化提示词
- **使用清晰的标题和分节**：将提示词分为多个明确的部分，如角色定位、抓取目标、要求等
- **使用编号列表**：使提示词层次清晰，易于理解
- **使用粗体和斜体**：突出重要信息和要求

#### 1.2 指令顺序优化
1. **角色定位**：首先明确AI的角色，建立专业背景
2. **任务目标**：清晰定义抓取或分析的目标
3. **具体要求**：详细列出具体的抓取或分析要求
4. **限制条件**：明确时间、数量等限制
5. **输出格式**：明确指定输出格式和字段
6. **平台特定策略**：针对不同平台的特殊处理

### 2. 关键词优化技巧

#### 2.1 关键词选择
- **核心技能关键词**：根据目标岗位选择最相关的核心技能
- **行业术语**：包含目标行业的专业术语
- **地理位置关键词**：精确到城市级别的地理位置
- **机构类型关键词**：明确目标机构类型

#### 2.2 关键词权重调整
- **高频关键词**：在提示词中多次出现的关键词会被赋予更高权重
- **关键词组合**：使用"技能 + 经验"、"学历 + 行业"等组合
- **否定关键词**：使用"不包括"、"排除"等词语排除不需要的岗位

### 3. 平台特定优化

#### 3.1 LinkedIn优化
- **使用精确的公司名称**：LinkedIn对公司名称匹配度要求高
- **包含职位级别**：如"Senior"、"Principal"等
- **利用LinkedIn高级搜索语法**：如"skill:Python"、"company:Google"

#### 3.2 Glassdoor优化
- **关注薪资信息**：Glassdoor提供丰富的薪资数据
- **包含公司评分**：利用Glassdoor的公司评分筛选高质量岗位
- **注意职位描述长度**：Glassdoor的职位描述通常较长，需要明确提取关键信息

#### 3.3 大学官网优化
- **关注院系结构**：大学岗位通常按院系分类
- **包含申请截止日期**：大学岗位通常有明确的截止日期
- **注意 tenure-track 信息**：学术岗位的 tenure-track 状态很重要

### 4. 性能优化

#### 4.1 抓取效率优化
- **合理设置抓取数量**：根据用户级别和需求设置合适的抓取数量
- **优化时间限制**：设置合理的抓取时间限制，避免超时
- **并行抓取策略**：对不同平台使用并行抓取，提高效率

#### 4.2 反爬虫策略
- **合理设置延迟**：在请求之间设置适当的延迟
- **随机化请求头**：模拟真实浏览器行为
- **使用代理**：对高频抓取使用代理IP
- **尊重robots.txt**：遵守网站的爬虫规则

### 5. A/B测试策略

#### 5.1 测试设计
- **变量定义**：明确要测试的变量，如关键词组合、提示词结构等
- **对照组设置**：设置基准提示词作为对照组
- **测试指标**：定义明确的测试指标，如抓取成功率、匹配度等

#### 5.2 测试执行
- **小批量测试**：先进行小批量测试，验证基本效果
- **大规模测试**：在小批量测试通过后进行大规模测试
- **数据收集**：收集详细的测试数据，包括抓取数量、质量、时间等

#### 5.3 结果分析
- **定量分析**：分析测试指标的数值差异
- **定性分析**：评估抓取结果的质量和相关性
- **综合评估**：综合考虑各种因素，选择最优方案

### 6. 提示词管理最佳实践

#### 6.1 版本控制
- **语义化版本号**：使用语义化版本号管理提示词版本
- **版本记录**：记录每个版本的修改内容和测试结果
- **回滚机制**：保留旧版本，以便在新版本效果不佳时回滚

#### 6.2 定期更新
- **季度更新**：每季度根据市场变化和用户反馈更新提示词
- **事件触发更新**：在招聘季节变化、行业趋势变化时及时更新
- **A/B测试驱动更新**：根据A/B测试结果持续优化

#### 6.3 反馈循环
- **用户反馈收集**：通过满意度评分和详细反馈收集用户意见
- **数据分析**：分析抓取和匹配数据，识别问题和优化机会
- **持续改进**：基于反馈和数据持续改进提示词质量

## 三、常见问题与解决方案

### 1. 抓取结果质量差
- **问题原因**：关键词不够精确，平台策略不合适
- **解决方案**：优化关键词组合，调整平台特定策略，增加质量筛选条件

### 2. 抓取速度慢
- **问题原因**：提示词过于复杂，抓取范围过大
- **解决方案**：简化提示词结构，缩小抓取范围，优化并行抓取策略

### 3. 匹配度低
- **问题原因**：用户简历信息不完整，提示词匹配逻辑不合理
- **解决方案**：完善用户简历信息，优化匹配算法，调整关键词权重

### 4. 重复岗位
- **问题原因**：去重逻辑不完善，不同平台同一岗位的表示方式不同
- **解决方案**：增强去重逻辑，基于岗位ID、URL、标题和公司名称的组合去重

### 5. 平台限制触发
- **问题原因**：抓取频率过高，请求模式过于规律
- **解决方案**：降低抓取频率，随机化请求间隔，使用代理IP
